# Project：从 Qwen3.5 到 KV Cache

## 项目总体情况

本仓库分为**多个阶段**，希望帮助同学系统理解**大语言模型推理**：在真实、较新的 **Qwen3.5** 架构上跑通生成，并逐步深入与效率、正确性相关的实现细节——其中 **KV Cache（键值缓存）** 将在后续 Phase 中作为核心数据结构反复出现。

| 阶段 | 主题方向（示意） |
|------|------------------|
| **Phase 1** | 配置环境、拉取权重，跑通统一基线命令 |
| 后续 Phase | 手写/对比 forward、增量解码、KV Cache 与混合缓存的更新与复用、与 Transformers 官方实现对齐等 |

**Phase 1 目标**：完成依赖与模型部署，执行下方命令流式看到模型输出；**无需改代码**，先建立「能跑起来」的基线。

进入仓库根目录后，**依次**执行以下两行（安装依赖、激活环境、按统一参数推理）：

```bash
uv sync && source .venv/bin/activate
chmod +x run.sh
./run.sh
```

- 首次运行会从 Hugging Face 下载 **Qwen3.5-0.8B**；如果卡住不动/下载极慢请检查你的网络环境问题。

## Qwen3.5-0.8B 与架构要点（实验载体选择原因）

**Qwen3.5** 是通义千问团队开源的**最新一代 Decoder-only Causal Language Model（因果语言模型）**，在开放权重模型中属于**最先进路线之一**。与当前主流大模型类似，Qwen3.5 遵循**「只看左侧上下文、逐步预测下一 token」**的因果推理范式。

**Qwen/Qwen3.5-0.8B** 是该系列中的**稠密小模型**（**无 MoE**），适合本地部署与课堂演示，并与更大规格、多模态成员**共享同一套设计思想**。

与常见的「**纯标准 Softmax 自注意力**」架构相比，Qwen3.5 具有以下**关键特点**：

1. **原生多模态设计（系列级共享架构）**  
   Qwen3.5 系列在**处理器、对话模板、视觉编码与语言模型的融合**上一体化设计，而不是「先训纯文本 LLM、再外挂视觉头」的简单拼接。**0.8B** 在本仓库中**主要承担文本推理**；该模型已支持**图像、视频**等多模态输入，体现**原生多模态兼容性**。

2. **混合注意力机制（继承 Qwen3-Next 思路）**  
   在层堆叠中，模型**交替**使用 **Full Attention（全注意力）** 与 **Linear Attention（线性注意力）** 模块：
   
   - 全注意力层采用 **RoPE** 与**标准 softmax 注意力**，实现中常配合**门控**（如对注意力输出施加 **sigmoid gate**）；
   - 线性注意力层则采用 **Gated Delta Net** 等结构，以**近似线性复杂度**换取对**长上下文**的扩展能力。

3. **门控与递推状态（强化表达能力）**  
   线性注意力通过**门控系数**与 **delta-style** 更新维持**跨时间步的内部状态**，与「单次写出 \(\mathrm{softmax}(QK^\top)V\)」的经典注意力有本质区别；思想上接近**带隐状态的序列模块**（如线性 RNN、**Mamba** 等），便于在长序列上**存取与更新历史信息**。分清 **全注意力层（侧重 KV Cache）** 与 **线性层（侧重状态递推与额外缓存）**，是理解 Qwen3.5 **推理机制**与本课 **KV / 混合缓存** 题目的关键。

**总结**：Qwen3.5 集 **原生多模态设计、混合注意力、门控递推子层** 与 **Causal LM 范式** 于一体，在高效文本推理的基础上具备扩展至多模态输入的潜力；其架构**适合作为本实验载体**，尤其在**长上下文**与**多模态**相关讨论中具有代表性。
